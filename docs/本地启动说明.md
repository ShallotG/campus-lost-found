# 智能校园失物招领平台 — 本地启动说明

## 一、环境要求总览

| 组件 | 版本要求 | 用途 |
|------|---------|------|
| JDK | **17** | 后端运行 |
| MySQL | **8.0** | 数据持久化 |
| Redis | **7.0** | 缓存 + JWT黑名单 + 限流 |
| Node.js | **18 LTS** | 前端开发服务器 |
| Python | **3.11** | AI检测服务 (YOLO) |
| Maven | 3.8+ | 后端构建 (IDE自带可跳过) |

> 如果本地没有 Python 环境，AI 检测功能会自动降级为手动填写，不影响系统运行。

---

## 二、第 1 步：数据库初始化

### 2.1 启动 MySQL

确保 MySQL 8.0 已启动。**如果使用 Docker 快速启动**：

```bash
docker run -d --name mysql-camp \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=123456 \
  mysql:8.0
```

### 2.2 建表 + 导入数据

**方式 A：命令行导入**
```bash
mysql -u root -p123456 < docker/mysql/init.sql
```

**方式 B：用 Navicat / DBeaver 等工具**
1. 连接 MySQL (root / 123456)
2. 打开 `docker/mysql/init.sql`
3. 执行全部语句

### 2.3 验证

```sql
USE campus_lost_found;
SELECT username, role FROM user;
-- 应输出：
-- admin   ROLE_ADMIN
-- staff01 ROLE_STAFF
```

---

## 三、第 2 步：启动 Redis

**Docker 快速启动**：

```bash
docker run -d --name redis-camp -p 6379:6379 redis:7-alpine
```

**验证**：
```bash
redis-cli ping
# 返回 PONG
```

---

## 四、第 3 步：启动 YOLO AI 服务（可选）

如果你的机器有 Python 3.11：

```bash
# 进入 ai-service 目录
cd ai-service

# 创建虚拟环境（推荐，避免污染全局）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 安装依赖（首次大约需下载 200MB 的 YOLO 模型）
pip install -r requirements.txt

# 启动服务（默认端口 5000）
python app.py
```

启动成功标志：
```
YOLO检测服务启动在端口 5000
正在加载YOLO模型: yolov8n.pt
YOLO模型加载完成
```

**如果不想装 Python / 没有 GPU**：
- YOLO 服务不可用时，上传图片后会返回类别 `"未知物品"`，工作人员**手动填写类别即可**，不影响核心流程。
- 在 `application.yml` 中设置 `ai.yolo.enabled: false` 可完全禁用。

## 五、第 4 步：配置关键参数

### 5.1 确认数据库连接

编辑 `backend/src/main/resources/application.yml`，确认你的 MySQL 账号密码：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_lost_found?...
    username: root
    password: 123456     # ← 改成你的密码
```

### 5.2 （可选）配置 DeepSeek API Key

如果要用 AI 语义检索功能，需要申请 DeepSeek API Key（[platform.deepseek.com](https://platform.deepseek.com)），然后填入：

```yaml
ai:
  deepseek:
    api-key: sk-xxxxxxxxxxxxxxxx    # ← 填你的 key
```

没有 API Key 时，检索会自动**降级为 MySQL 关键词匹配**，也能正常使用。

### 5.3 （可选）配置 OSS

如果要上传文件到阿里云 OSS 而非本地存储：

```yaml
file:
  storage-mode: oss
  oss:
    endpoint: oss-cn-hangzhou.aliyuncs.com
    access-key-id: LTAI5txxxx
    access-key-secret: wJm9xxxx
    bucket-name: campus-lost-found
```

不配则默认存本地 `./uploads/` 目录。

## 八、第 7 步：访问验证

打开浏览器访问：

| 地址 | 说明 |
|------|------|
| `http://localhost:3000` | 失主端首页（自动跳转登录） |
| `http://localhost:3000/user/login` | 失主登录页 |
| `http://localhost:3000/user/register` | 失主注册页 |
| `http://localhost:3000/admin/login` | 管理端登录页 |

### 默认账号

| 角色 | 用户名 | 密码 |
|------|--------|------|
| 管理员 | admin | admin123 |
| 工作人员 | staff01 | staff123 |
| 失主 | 自行注册 | - |

### 端到端验证流程

1. 浏览器打开 `http://localhost:3000/admin/login`
2. 用 **staff01 / staff123** 登录管理端
3. 点击左侧「拾物登记」→ 上传一张物品图片 → AI 自动识别类别 → 填写存放位置 → 提交
4. 新窗口打开 `http://localhost:3000/user/register` → 注册一个失主账号
5. 登录失主端 → 在检索页输入描述文字 → 点击「智能检索」
6. 看到匹配结果 → 点击「确认是我的物品」→ 弹出招领处联系方式
7. 回到管理端 → 拾物列表 → 找到该物品 → 点击「领取」→ 登记领取人信息
8. 查看「数据看板」→ 统计数据更新

## 九、快速启动汇总（纯 Docker）

如果已配好 Docker，可以一键启动 MySQL + Redis：

```bash
# 仅启动基础服务
docker run -d --name mysql-camp -p 3306:3306 -e MYSQL_ROOT_PASSWORD=123456 mysql:8.0
docker run -d --name redis-camp -p 6379:6379 redis:7-alpine

# 等 MySQL ready（约 30 秒），导入数据
sleep 30
docker exec -i mysql-camp mysql -uroot -p123456 < docker/mysql/init.sql
```

然后只需 3 个终端分别启动：

```bash
# 终端 1: AI检测（可选）
cd ai-service && python app.py

# 终端 2: 后端
cd backend && mvn spring-boot:run

# 终端 3: 前端
cd frontend && npm install && npm run dev
```

## 十、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 后端启动报 `Communications link failure` | MySQL 未启动或密码不对 | 确认 MySQL 服务运行中，检查 `application.yml` 密码 |
| 登录失败 `401` | 数据库未初始化 | 执行 `docker/mysql/init.sql` |
| 上传图片后类别显示"未知物品" | YOLO 服务未启动 | 启动 `ai-service` 或手动填写类别 |
| 检索返回结果为空 | DeepSeek Key 未配置 | 改为关键词匹配模式，不影响使用 |
| 前端报 `Network Error` | 后端未启动 | 确认 `localhost:8080` 可访问 |
| `npm install` 卡住 | npm 源慢 | 设置镜像：`npm config set registry https://registry.npmmirror.com` |
| YOLO 首次启动下载模型很慢 | 网络原因 | 可手动下载 `yolov8n.pt` 放到 `ai-service/model/` 目录 |
